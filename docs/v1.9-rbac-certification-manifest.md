## Certified capabilities

v1.9 provides a narrow role-based authorization layer for the existing read-only Nexus Admin API.

- Centralized RBAC at the Admin API HTTP boundary.
- Existing `X-Nexus-Admin-Key` authentication retained.
- Existing `AdminKey` remains backward-compatible and grants Admin access.
- Optional `AuditorKey` and `ViewerKey` configuration entries.
- `/health` remains open.
- `/trust/{module}` requires Viewer or higher.
- `/dqs` requires Auditor or higher.
- `/audit` requires Auditor or higher.
- Unknown protected endpoints default to Admin.
- `401 unauthorized` for missing or invalid keys.
- `403 forbidden` for valid keys with insufficient role.
- Environment-variable deployment guidance for role keys.
- CI-enforced xUnit coverage for role resolution, hierarchy, placeholder handling, and endpoint requirements.
- Tests exercise the shared production authorization implementation, not duplicated test-only logic.

## Governance boundaries

v1.9 does **not** include:

- JWT authentication.
- SSO or external identity provider integration.
- User account persistence.
- Per-user sessions.
- Dashboard authentication.
- Write-side Admin API behavior.
- Broader policy engine changes.
- Simulation behavior changes.
- Persistence schema expansion.

## Completion decision

v1.9 Role-Based Authorization is complete.

Further authentication work must begin as a new, separately scoped security phase. It must not be added as an unplanned continuation of v1.9.

Recommended next phase boundary:

- v1.10 Authentication Architecture and Identity Integration Planning.
